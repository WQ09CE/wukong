# Wukong Demo Guide (演示指南)

> 5-10 分钟快速演示 Wukong 多智能体协作系统

## 演示前准备

### 1. 环境准备

```bash
# 确保已安装 wukong
cd /path/to/demo-project
wukong doctor  # 检查安装状态

# 启动 Claude Code
claude
```

### 2. 准备一个简单的演示项目

可以用任意 Python/JS 项目，或者临时创建：

```bash
mkdir demo-project && cd demo-project
git init
echo "# Demo Project" > README.md
mkdir src
echo 'def hello(): return "hello"' > src/main.py
```

## 演示流程 (推荐顺序)

### Demo 1: 激活悟空 (30秒)

**输入:**
```
/wukong 你好
```

**展示点:**
- 悟空响应，说明系统已就绪
- 介绍六根分身概念

---

### Demo 2: 眼分身探索 (1分钟)

**输入:**
```
/wukong @眼 探索这个项目的结构
```

**展示点:**
- 显式 `@眼` 语法直接召唤眼分身
- 眼分身**后台运行**，不阻塞
- 输出文件列表和项目发现

**讲解:**
> "眼分身负责探索和搜索，它只能读取，不能修改代码。这是职责边界的体现。"

---

### Demo 3: 意分身设计 (1-2分钟)

**输入:**
```
/wukong @意 设计一个简单的用户认证模块，包含登录和注册
```

**展示点:**
- 意分身输出架构设计
- 包含接口定义、技术选型、权衡分析
- 意分身**只设计，不写代码**

**讲解:**
> "意分身是架构师，它做设计决策，但实现交给斗战胜佛。"

---

### Demo 4: 斗战胜佛实现 (2-3分钟)

**输入:**
```
/wukong @斗战胜佛 实现一个简单的 add 函数，接收两个数字返回和
```

**展示点:**
- 斗战胜佛写代码
- 自动运行验证（语法检查）
- 输出修改的文件列表

**讲解:**
> "斗战胜佛是唯一能写大量代码的分身。本体只做调度，不直接写代码。"

---

### Demo 5: 自动轨道选择 (1-2分钟)

**输入:**
```
/wukong 添加一个 subtract 函数
```

**展示点:**
- 没有 `@` 指定，系统自动选择轨道
- 检测到 "添加" → Feature Track
- 自动编排分身流程

**讲解:**
> "不指定分身时，系统根据关键词自动选择工作流。添加功能走 Feature 轨道，修复 bug 走 Fix 轨道。"

---

### Demo 6: 筋斗云并行 (1-2分钟)

**输入:**
```
/wukong @斗战胜佛 同时在 src/math.py 添加 multiply 函数，在 src/string.py 添加 reverse 函数
```

**展示点:**
- 两个独立文件 → 并行召唤两个分身
- 效率提升 2x

**讲解:**
> "筋斗云机制：独立任务并行执行，不把多个任务打包给一个分身串行处理。"

---

### Demo 7: 戒定慧识验证 (1分钟)

**输入:**
```
/wukong @鼻 审查 src/main.py
```

**展示点:**
- 鼻分身审查代码
- 输出问题列表和严重程度
- 鼻分身**只审查，不修复**

**讲解:**
> "戒定慧识是验证闭环：戒检查规则，定验证可复现，慧提取锚点，识存储反馈。"

---

### Demo 8: 锚点系统 (30秒)

**输入:**
```
/wukong 锚点
```

**展示点:**
- 显示项目中的决策锚点 [D]、约束锚点 [C]、问题锚点 [P]
- 锚点跨会话保留

**讲解:**
> "锚点是不可压缩的关键信息，无论上下文如何缩小，锚点始终保留。"

---

## 一句话总结

> **"六根并行生产，戒定慧识护航，筋斗云加速，锚点永久沉淀。"**

## 常见问题 Q&A

**Q: 和普通 Claude Code 有什么区别？**
> A: 普通 Claude 是单线程思考，Wukong 是多分身并行 + 职责隔离 + 验证闭环。

**Q: 为什么要分这么多分身？**
> A: 职责隔离。眼分身只探索不修改，意分身只设计不实现。防止 AI "既当裁判又当运动员"。

**Q: 锚点有什么用？**
> A: 跨会话记忆。上次做的决策、踩的坑，下次不用重新解释。

**Q: 筋斗云并行能提升多少效率？**
> A: 独立任务 2-4 倍。比如修改 3 个独立文件，串行 3 分钟，并行 1 分钟。

## 演示时的小技巧

1. **先展示简单的** - 从 `@眼` 探索开始，再到 `@斗战胜佛` 实现
2. **强调职责边界** - 每个分身有明确的 Do/Don't
3. **对比效果** - 可以先展示普通模式，再展示 Wukong 模式
4. **用真实项目** - 如果有时间，用观众熟悉的项目演示更有说服力
